# 進階：文獻知識庫與 Obsidian 整合（選配）

這份文件說明如何把七道工序的文獻產出，累積成一個跨專案、可長期複用的文獻知識庫。它是選配的增強層，不是核心工作流的必要條件：只用控制檔、資料夾治理與七支 skill，你已經可以完成一篇論文的文獻工作。但如果你希望「這一篇分析過的文獻，下一篇不必重讀」，並且想一眼看出「哪些論文引用過同一篇文獻」，這一層會把前面幾道工序的價值放大。

以下依文獻在工作流裡的實際行進順序說明，從蒐集到建庫。

## 一、蒐集：Zotero 與文獻池

文獻搜尋工序（literature-search）以 API 產生候選文獻池、逐筆驗證 DOI，最後輸出一份 DOI 清單 `zotero_import_dois.txt`。你在本機打開 Zotero，用「透過識別碼新增」（Add by Identifier）貼入這批 DOI 批次匯入，Zotero 會盡力抓取 PDF；抓不到的依清單提示手動補齊。PDF 由你在本機取得，工作流不代抓，這同時是技術限制與著作權邊界。取得的 PDF 先放進當前論文專案的 `01_文獻/`，供下一道工序讀取。

Zotero 在整套設計裡的角色不只是抓 PDF，它為每篇文獻提供一個穩定的 `citation_key`，這個鍵之後會成為知識庫與 Zotero 之間的橋梁。

## 二、分析：單篇雙檔

逐篇分析工序（literature-analysis）對每篇 PDF 產出兩個檔案。知識檔記載這篇文獻本身說了什麼，與你的任何論文無關，可跨論文複用；應用檔記載它對你當前這篇論文有什麼用，綁定論文編號。這個雙檔分離是知識庫能長期累積的前提：知識沉澱一次寫成，換一篇論文只需補一份新的應用檔。

兩種檔案最前端都帶 YAML frontmatter，讓同一份 `.md` 同時滿足三端：工作流端供綜整工序解析、建立文獻清單（不限執行環境，claude.ai 上的 skill 一樣讀 frontmatter），Obsidian 端支援 graph view 與 backlink，Zotero 端以 `citation_key` 連回原始條目。

## 三、綜整：跨篇

跨篇綜整工序（literature-synthesis）把多篇分析整合為帶明確文獻歸屬的理論知識圖譜，同樣輸出綜整知識檔與綜整應用檔。到這一步，單篇與跨篇的知識都已成形，接著才進入知識庫。

## 四、晉升進 vault

所有工序的產出都先落在當前專案的 `05_輸出/`。你讀過、確認品質無誤後，親手把定稿移入文獻庫 vault。AI 不直接寫入 vault，晉升這個動作是你的簽核，也是保護知識庫正本的最後一道關卡。docx 合併版（知識與應用接續）留在論文專案夾供 Word 閱讀批注，不進 vault。

## 五、知識庫的建構與運作

vault 採「單一資料夾三位一體」：同一個資料夾同時是你的人工文獻庫、Obsidian vault、與 Zotero 附件庫，不必維護第二套。路徑由你自訂，例如放在雲端同步夾下的 `3_文獻/`。

佈局上，知識檔存在分類樹下，例如 `理論基礎/[主題]/`；PDF 晉升進 vault 時改名成與知識檔同前綴的 `[主題]_[作者年份]_[關鍵詞].pdf`，與知識檔並排同夾。應用檔存在 `應用/[論文編號]/`。vault 根目錄放一份索引 `_index.json`，記錄每篇文獻的主題、檔案位置、以及被哪些論文引用過。

連結是這一層的關鍵，且刻意做成與資料夾位置無關（folder-independent）：跨檔連結一律用 Obsidian 的 basename wikilink，不寫資料夾路徑，日後重組資料夾連結也不會斷。其中兩條連結最重要。知識檔標題下一行放 `> 📎 原文 PDF：[[主題_作者年份_關鍵詞.pdf]]`，讓你在 vault 內一鍵開啟 PDF；應用檔內文放 `> 🔗 對應知識檔：[[..._知識]]`，這一行才會驅動 Obsidian 的反向連結（backlink）。frontmatter 裡的 `pdf_path`、`knowledge_note` 僅作 metadata，不會自動生成可點連結。

## 為什麼值得

雙檔分離，加上那條 `[[..._知識]]` wikilink，換來的是知識庫能自動回答一個平常很難追蹤的問題：哪些論文引用過同一篇文獻。你打開任一篇知識檔，Obsidian 的 backlink 面板就列出所有用過它的應用檔，也就是所有用過這篇文獻的論文。文獻愈積愈多、論文愈寫愈多，這個跨專案的引用網絡就愈有價值，這正是雙檔分離設計在單篇層次看不出、要到知識庫層次才兌現的回報。

再次提醒，這一層是選配。它需要你採用 Obsidian 與 Zotero 這兩個特定工具；不用它們，核心工作流一樣運作，你只是少了跨專案自動累積與反向連結這個增強。是否值得，取決於你是否有長期、跨多篇論文的文獻複用需求。
