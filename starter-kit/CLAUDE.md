# 工作區規則（AI 開場即讀）

本檔在工作區被掛載時自動載入，是本工作區資料夾治理規則的常駐版本。**不論是否觸發任何 skill**，只要在這個工作區內動檔案，以下規則一律適用。與各 skill 的 SKILL.md 衝突時，以 SKILL.md 的執行協議為準；SKILL.md 未涵蓋之處，以本檔為準。

## 1. 讀寫分區

| 資料夾 | 讀寫規則 |
|---|---|
| `00_專案控制/` | 唯讀。四份控制檔（`about-me.md`、`research-identity.md`、`writing-standards.md`、`portfolio-status.md`）與思考層中樞 `paper-structure.md`。任務執行時只讀不寫。 |
| `01_文獻/` | 唯讀。文獻 PDF 與分析定稿，晉升由使用者手動執行。 |
| `02_資料與證據/` | 唯讀。一手資料（訪談逐字稿、田野筆記、數據）。 |
| `03_寫作/` | 唯讀。章節定稿，由使用者手動移入。 |
| `04_審稿與回應/` | 唯讀。審稿意見與回應。 |
| `05_輸出/` | **唯一寫入區**。AI 的所有產出一律落在此處，依專案建子資料夾分流。 |

**唯讀的唯一例外**：literature-analysis 依其 SKILL 協議回寫 `00_專案控制/paper-structure.md` 的「文獻分析完成清單」。此為已授權例外，不需另行詢問。除此之外，不得在 00–04 新增、修改或刪除任何檔案。

**不刪除任何檔案。** 需要刪除時一律先問使用者。

## 2. 晉升流程（產出、審定、晉升）

AI 生產，人蓋章，蓋章的動作就是把檔案搬過那條唯讀線：

1. 所有產出先落 `05_輸出/`。
2. 使用者讀過、確認品質無誤。
3. 由**使用者手動**把定稿搬入正式位置（分析定稿 → `01_文獻/`；章節定稿 → `03_寫作/`；`paper-structure.md` → `00_專案控制/`）。

對唯讀區既有檔案的修訂版（例如 `paper-structure.md` 的改版），同樣產出於 `05_輸出/`，由使用者確認後手動覆蓋，不得直接改寫正本。任務開始時若發現 `05_輸出/` 有比正式位置更新的同名檔案，主動提示使用者有未晉升的修訂版，確認以哪一版為準後才繼續。

## 3. 檔案命名

格式：`專案名稱_內容類型_YYYYMMDD_v1.副檔名`
同日同名檔案已存在時，版本號遞增（v2、v3……）。各 skill 的產出檔名若其 SKILL 協議另有規定，依 SKILL 協議。

## 4. 品質底線

- 未達 `00_專案控制/writing-standards.md` 標準的段落，**交付並標記**「此段需要修正，原因是：」，而不是扣住不交付。
- 推論強度必須與研究設計相符：橫斷面資料不使用強因果語氣、質性個案不宣稱統計代表性。
- **文獻真實性**：任何產出中不得出現未經 literature-analysis 審核、或未經使用者明確確認的文獻引用。不確定的引用一律標記【待查證】，不直接寫入正文。

## 5. 誠實聲明

以上唯讀分區由本檔與各 skill 條文執行，**並非作業系統層的硬性權限**。真正不可逆的保護來自雲端同步資料夾的版本歷史，以及「人工搬檔＝簽核」的晉升紀律。
