# 01_文獻/

**用途**：存放文獻搜尋池、逐篇文獻分析檔（literature-analysis 產出）、跨篇綜整檔（literature-synthesis 產出）等與文獻相關的知識性資料。

**讀寫紀律**：任務執行時唯讀。AI 在協作過程中會讀取本夾內容作為分析與寫作依據，但不會直接寫入本夾；文獻分析／綜整的新產出應寫入 `05_輸出/`，經你確認後再視情況搬入本夾定案。

**與文獻 vault 的關係**：literature-analysis／literature-synthesis 兩道工序以「文獻 vault」（一個由你自管的文獻庫資料夾，skill 內慣例名 `3_文獻`，可用 Obsidian 開啟；實際路徑與名稱由你自訂）作為知識檔／綜整檔審定後的長期正本落點。最簡單的做法是直接把本夾當作 vault 根目錄：在夾內建立 `理論基礎/`（依主題存放知識檔與對應 PDF）與 `應用/`（依論文存放應用檔）兩個子樹即可。不使用 Obsidian 也不影響工作流，vault 相關敘述可視為進階選項。

**檔名規則**：`專案名稱_內容類型_YYYYMMDD_v1.副檔名`（如 `paper1_文獻池_20260714_v1.md`）；同名檔案更新時遞增版號（v2、v3…），不覆蓋舊版。
