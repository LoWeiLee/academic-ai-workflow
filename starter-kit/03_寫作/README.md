# 03_寫作/

**用途**：存放章節草稿與正文等寫作過程檔案。（研究設計骨架 `paper-structure.md` 的正本放在 `00_專案控制/`，它是各工序讀取與回寫的錨點，不在本夾。）

**讀寫紀律**：任務執行時唯讀。research-design-diagnosis、chapter-drafting 等 skill 會讀取本夾既有結構與草稿作為協作基礎，但新的章節產出或修訂稿一律先寫入 `05_輸出/`，經你確認後再手動搬入本夾更新定案版本。

**檔名規則**：`專案名稱_內容類型_YYYYMMDD_v1.副檔名`（如 `paper1_第二章草稿_20260714_v1.docx`）；同名檔案修訂時遞增版號，不直接覆蓋既有檔案。
