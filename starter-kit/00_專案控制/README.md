# 00_專案控制／控制檔說明

本夾是整套工作流的規則層：四份控制檔＋一份由工序一產出的 `paper-structure.md`。各 skill 啟動時從這裡讀取「你是誰、你的研究長什麼樣、寫作紀律是什麼」。

**四份控制檔（四份就位、兩份必填）**
- `about-me.md`（必填）：研究發展目標、優先序規則、AI 角色設定、寫作慣性自我申報
- `research-identity.md`（必填）：研究定位與理論版圖，按掌握深度分五級
- `writing-standards.md`（沿用預設即可）：禁止清單、引用格式、學術中文語氣
- `portfolio-status.md`（可留空）：發表組合動態儀表板，多線並行時再補

**`paper-structure.md`**：初始不存在是正常的——它由工序一（research-design-diagnosis）產出、落在 `05_輸出/`，經你審定後改名搬入本夾（「正本化」）。它是工序 2–5 的路徑錨點。

**讀寫紀律**：本夾在任務執行時唯讀，AI 只讀取不寫入；唯一例外是 literature-analysis 依其協議回寫 `paper-structure.md` 的「文獻分析完成清單」。控制檔的修訂版一律先落 `05_輸出/`，由你確認後手動搬入本夾覆蓋。

**檔名規則**：控制檔用固定檔名（如上），不加日期與版號——skill 以固定路徑尋找它們。
