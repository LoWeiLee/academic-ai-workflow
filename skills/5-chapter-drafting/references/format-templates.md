# chapter-drafting｜逐字輸出模板（format-templates）

> 載入時機：執行撰寫循環時。SKILL.md 守決策層（五階段流程、自由度邊界、引用紀律、觸發閾值、介面合約），本檔提供逐字輸出骨架。範本與本檔衝突時：格式、語調、深度以範本為基準；規格與流程以 SKILL 條文為準。

---

## Part 0.3｜材料清點與啟動宣告

讀取完畢後，Skill 必須以下列格式宣告啟動狀態，**不得略過**：

```
【chapter-drafting 啟動就緒】
目標章節：[章節編號與名稱，例：第二章 2.2 節]
撰寫模式：[從零撰寫 / 既有草稿重寫 / 部分補強]
自由度設定：中等（段落骨架 + 推薦句型 + 引用建議）
互動模式：漸進式（寫一段，確認一段）
引用來源白名單：
  - 綜整知識檔：[檔名]（提供 N 個概念節點）
  - 綜整應用檔：[檔名]（5.2 引用配置 M 篇、5.4 概念使用對照 K 項）
  - paper-structure.md 已列文獻：[條列]
  - 既有 argument-bank.md：[若有則列出條目數]
本節預計段落數：[依 paper-structure.md 的論述邏輯推估，例：4-6 段]
專案品質閘門：[已載入 folder-instructions.md 第 I 部分 N 條閘門 / 本專案無 folder-instructions.md，專案專屬閘門 0 條，僅以 00_專案控制/ 四份控制檔為準]
範本狀態：[已載入 example-chapter-drafting.md / 尚無範本，本次產出為候選範本]
語感基準：[已載入 writing-standards.md 使用者寫作風格庫 N 項可模仿語感 / 該檔無風格庫，改走通用語氣標準]

請使用者確認以上設定，或指示調整後，進入 Part 1 的段落骨架階段。
```

## Part 1 階段 2｜段落骨架格式

```
【段落 N 骨架】

首句主張（使用者決定具體措辭，骨架只給方向）：
  〔此處寫 Claude 建議的主張方向，一至二句〕

論述步驟：
  步驟 1｜[步驟名，例：建立上位概念]
  ├─ 引用候選：[從綜整檔取出的 1-2 個可引用節點，含頁碼]
  └─ 銜接句型候選（選一）：
      A) 「在 [領域] 研究中，[概念] 被定義為……」
      B) 「[學者] (年份) 將 [概念] 界定為……，作為……的分析起點」
      C) 其他（使用者自擬）

  步驟 2｜[步驟名，例：處理概念爭議 / 引入限定條件]
  ├─ 引用候選：[從綜整檔取出的節點]
  └─ 銜接句型候選（選一）：
      A) 「然而，這個定義面臨 [挑戰]……」
      B) 「值得注意的限定是……」
      C) 其他

  步驟 3｜[步驟名，例：收束回本研究]
  ├─ 引用候選：[若有]
  └─ 銜接句型候選（選一）：
      A) 「對本研究而言，這意味著……」
      B) 「據此，本文採取 [選擇]……」
      C) 其他

結語收束（本段最後一句的功能）：
  [例：為下一段鋪陳「多重觀點並存」的必要性]

預計字數：[依 paper-structure.md 估算，例：250-350 字]
```

## Part 1 階段 4｜填充稿格式

```
【段落 N 填充稿】

〔完整段落文字，符合學術中文期刊慣例，無粗體、無副標、無 emoji〕

〔若段落內有引用，直接以 APA 7th 格式嵌入，例：Wu et al.（2015, p. 166）指出……〕

---
使用的引用清單：
  - [作者 (年份), 頁碼]：[此引用在本段的具體功能]
  - [作者 (年份), 頁碼]：[功能]

本段自檢（內部標記，不入正文）：
  <!-- ✓ 首句承載核心主張 -->
  <!-- ✓ 引用具備支持/對比/補充/挑戰功能 -->
  <!-- ✓ 與 paper-structure.md 論述邏輯一致 -->
  <!-- ✓ 推論強度與研究設計相符 -->
  <!-- ✓ Part 2.6 claim 稽核：每一實質論斷已分類（描述/詮釋/因果規範）並錨定來源，無 [待證] -->
  <!-- ✓ 無 writing-standards.md 禁止清單內的寫作痕跡 -->
  <!-- ✓ 語感對齊 writing-standards.md 使用者寫作風格庫（若已建立），未複製使用者防缺陷項（依你的風格庫設定，如超長句／特定虛詞過量等） -->
  <!-- ✓ 通過 folder-instructions.md 第 I 部分各專案閘門（逐條，如適用） -->

若上述任一項自檢失敗，於段落上方以 **[此段需要修正，原因是：XXX]** 標記，不交付通過。
```

## Part 2.2｜白名單外文獻警示

```
【白名單外文獻需求警示】

目前段落 N 的步驟 M 需要一個能支撐 [XX 論點] 的引用，但白名單內無合適文獻。
建議選項：

A. 暫停本段填充，先啟動 literature-analysis 分析 [候選文獻]，完成後加入白名單再回來填充
B. 改用白名單內的 [替代文獻]，但論點強度會從「直接支持」降為「間接呼應」
C. 使用者明確授權先行引用 [候選文獻]，但須標記「待補 literature-analysis 分析」
D. 刪除此步驟，重新設計論述邏輯
E. 本需求源自無具名文獻的記憶宣稱（Part 2.7 導向）：刪除該宣稱，或降級為作者論點（詮釋性 claim，明標）——A–C 不適用（無具名文獻可分析）

請使用者選擇。
```

## Part 3.1｜文獻回顧擴張警示（機制範本一）

```
【文獻回顧擴張警示】（writing-standards.md 機制範本一）

目前 [章節] 累積字數 XX 字，已達 paper-structure.md 估算上限的 XXX%。
或：本段引用了 N 篇文獻，未呈現綜整性分析。

建議處置：
- 目前文獻量已可支撐 [研究缺口/論點 X]，建議在此設立邊界，轉入下一步
- 若要加入 [新文獻]，需評估其是否替代某個既有引用（而非疊加）
- 請使用者確認是否接受當前邊界。
```

## Part 3.2｜後設思維漂移警示（機制範本二）

```
【後設思維漂移警示】（writing-standards.md 機制範本二）

目前段落論述已上升至後設層次（[具體觀察]）。
建議暫停，先確認：
- 本段的具體論點是什麼？
- 這個後設層次的論述是否為本章必要？
- 若保留，可以如何用一句話收束回本章的具體任務？
```
