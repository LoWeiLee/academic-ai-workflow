# 審稿診斷技能 [現行版，見檔尾版本區]

> 本檔守決策層（定位、嚴格度、Pre-flight、評等規則、三階段流程判斷）。細節清單與逐字輸出模板下放 `references/`，於對應階段才載入：
> - `references/method-checklists.md` —— 第一階段五面向問題集、第二階段各章評估重點（含質性/個案/混合）、形式品質檢查（第一、二階段載入）
> - `references/output-templates.md` —— 第一階段診斷格式、Mode A/B/C 第三階段輸出模板、Mode D 表格與 Response Letter（產出階段載入）
> - `references/reviewer-self-audit.md` —— 三鏡頭、八類審稿者偏誤、校準五題（A/B/C 第三階段前載入）
> - `references/score-trajectory.md` —— 五面向跨輪 delta 與回歸偵測（複審/Mode D 載入）
> - `references/mode-e-disclosure.md` —— Mode E：期刊 AI 政策查核與聲明範本（Mode E 載入）
> - `references/reasoning-framework.md` —— 論證與推論框架，四支共用副本（細審載入）
>

## 核心定位

review-diagnosis 是研究 pipeline 中唯一不直接服務於「產出期刊」工作流的工具（橫切 QA，不在線性管線上）。它有四個獨立的使用目的：

**Mode A：碩博士論文口試審查**
協助使用者擔任碩士或博士論文口試委員時，提供結構化的審查支架，產出符合學位論文層級的口試委員審查意見表。

**Mode B：外部期刊稿件審查**
協助使用者擔任 TSSCI（一級/二級/三級）或國際 SSCI 期刊審稿人時，提供結構化的審查支架，產出符合期刊規範的正式審稿意見。

**Mode C：投稿前自我審查**
對自己已完成的稿件執行「強制陌生化」的反向診斷——用資深審稿人的骨架評估框架重讀自己熟悉的文章，找出作者視角容易忽略的結構性問題，產出修改優先序清單。

**Mode D：針對外審意見修正**
拿到外審意見後，輔助使用者確認需修正之處、研擬具體修正策略、製作逐點回應表格，並將修正整合進 Response Letter。此模式不執行品質診斷，而是執行「外審意見的結構化處理」。

**Mode E：投稿前 AI 使用揭露**
依目標期刊政策產生 AI 使用聲明草稿（不做品質診斷或外審回應）。獨立工作流，細節載入 `references/mode-e-disclosure.md`。

A、B、C 三個模式共用三階段框架，在第三階段分流輸出。Mode D、Mode E 各採獨立工作流。

## 嚴格度層級表

評估的嚴格度依稿件類型而異，影響「優/良/可」評等的門檻設定，以及修改建議的尺度。

| 嚴格度層級 | 稿件類型 | 對應模式 | 評等門檻 |
|-----------|---------|---------|---------|
| 最高 | 使用者自己稿件 | Mode C | 對等 TSSCI 一級 |
| 高 | TSSCI 一級期刊稿件 | Mode B | 理論貢獻必須明確 |
| 高 | 國際 SSCI 期刊稿件 | Mode B | 理論貢獻必須明確，且須與國際文獻充分對話 |
| 中高 | TSSCI 二級期刊稿件 | Mode B | 理論對話充分即可 |
| 中 | TSSCI 三級期刊稿件 | Mode B | 系統性整合即可 |
| 中低 | 博士論文 | Mode A | 重於方法嚴謹度與文獻完整性 |
| 較低 | 碩士論文 | Mode A | 達到學位論文最低水準即可 |
| — | Mode D | 外審意見處理 | 不適用此維度 |
| — | Mode E | AI 使用揭露 | 不適用此維度 |

**規則**：同樣是「優」評等，TSSCI 一級的標準遠高於碩士論文。在第一階段五面向評等時，必須先錨定本案的嚴格度層級，再決定門檻。

> **在地示例說明**：TSSCI（臺灣社會科學引文索引）分級為臺灣在地示例。非臺灣體系的使用者請以所屬學術體系的期刊分級替換（如 CSSCI、KCI 或 SSCI 分區），對應原則：一級≈該體系頂尖層、二級≈中堅層、三級≈一般收錄層，嚴格度門檻同表對應。

## 環境偵測（所有讀取動作之前，最先執行）

本 skill 以 `00_專案控制/` 作為控制檔的預設路徑，此路徑假設一個**持久的工作區資料夾**（Claude 桌面應用的 Cowork 模式，或 Claude Code）。**若該路徑不存在，代表使用者在無工作區環境（例如 claude.ai 網頁版）執行本 skill。此時一律進入「無工作區模式」，不得以「檔案缺失」為由拒絕啟動。**

無工作區模式的三條規則：

1. **控制檔改在別處尋找**。依序查找同名檔案：對話附件 → 專案知識庫（claude.ai 的 Project files）→ 使用者於本次對話明確指定的位置。找到即視同滿足必讀要求，正常推進；三處皆無才回報缺失，並明確告知使用者可透過對話附件或 Project files 提供。**此規則覆蓋本 skill 所有「缺漏即拒絕啟動」的硬閘門條文**——閘門的實質要求是「讀得到該檔的內容」，不是「該檔位於特定路徑」。
2. **產出改以可下載檔案交付**，不嘗試寫入 `05_輸出/` 或任何工作區路徑。晉升流程（審定後移入正式位置）改由使用者手動完成，於交付時一併說明。
3. **跨 session 狀態改由使用者提供**。本 skill 若依賴 `05_輸出/progress-log.md`、既有產出檔或凍結狀態回讀斷點，在無工作區模式下改為請使用者於開場提供前輪產出（附件或貼上內容）；無法取得時，明示「本次為無工作區模式，跨對話狀態不自動續作」後，以本次提供的材料為準推進。

**啟動宣告中必須載明當前模式**：「工作區模式」或「無工作區模式」。

## Pre-flight 必要確認

**在任何分析動作之前，必須先完成以下確認。若使用者已在指令中說明部分項目，直接確認並進入下一項。**

### 共用詢問項（所有模式）

**問項一：審稿模式為何？**
- Mode A：碩博士論文口試審查
- Mode B：外部期刊稿件審查
- Mode C：投稿前自我審查
- Mode D：針對外審意見修正
- Mode E：投稿前 AI 使用揭露

### 模式專屬詢問項

**Mode A 專屬**：
1. 學位類型：碩士論文 / 博士論文
2. 學程屬性：學術型（一般所） / 實務型（在職專班、EMBA、碩士學程）
3. 口試階段：初試 / 複試 / 二次口試（若是二次口試，須請使用者提供前次審查意見）

**Mode B 專屬**：
1. 期刊層級：TSSCI 一級 / TSSCI 二級 / TSSCI 三級 / 國際 SSCI（請說明具體期刊名稱）
2. 審查輪次：初審 / 複審 / 三審。**複審／三審必要輸入（S-0 基線）**：請使用者提供前一輪審查報告（`05_輸出/` 路徑或貼上前輪五面向評等表），或由 `05_輸出/progress-log.md` 前輪條目回讀；皆無＝明示「無基線，本輪不產分數軌跡」，禁止憑記憶重建前輪評等。

**Mode C 專屬**：
1. 目標期刊（決定評估的嚴格度錨點）
2. 預定投稿時程（影響修改建議的優先序）
3. **必讀：當前專案 `paper-structure.md`**——用以核對稿件的結論宣稱強度是否與研究設計、貢獻聲明一致（例如探索性框架是否被寫成已證實、橫斷面是否被寫成因果）。第一階段面向五與第二階段第 5 章評估時以此為對照基準。
4. **必讀：`writing-standards.md`**——將其禁止清單納入 Mode C 的檢查維度（空洞貢獻宣稱、因果語氣濫用、AI 寫作痕跡、質性論述典型問題等），於形式品質檢查與五面向診斷中一併比對。
5. **選讀：chapter-drafting 自檢註解**——受審章節若由 chapter-drafting 產出，讀取章節檔尾的 Part 7 自檢註解（`<!-- -->` 區塊）與小節署名，作為審稿脈絡並避免重複檢查已驗證項目。

> Mode A、B 對「他人稿件」審查，通常無法取得對方的 paper-structure.md，故此兩項必讀僅 Mode C（自審）強制；若 Mode B 審查的是使用者自己投稿前的再確認，亦可比照 Mode C 加讀。

### 共用次題（除 Mode D）

**問項二：方法論類型為何？**
- 量化研究 / 質性研究 / 混合方法

**允許分支**：若使用者不確定（常見於 Mode A 拿到稿件初次接觸），可選擇「待你看完先回判」。此時跳過此題，由 AI 在第一階段判讀後回報，再進入第二階段。

**問項三：是否需要對齊既有審查範本？**
- 是：請告知範本檔案位置（資料夾或具體檔名），AI 將先讀取既有範本，對齊格式、結構、語調、深度（範本與 SKILL 衝突時：格式、語調、深度以範本為基準；規格與流程以 SKILL 條文為準）
- 否：採用本 skill 預設輸出格式

**這三項資訊決定**：理論貢獻的評估門檻（嚴格度層級）、信效度的評估標準（方法論類型）、以及第三階段的輸出格式（審稿模式 + 範本對齊）。

## 五面向評等系統

本 skill 採「五面向評等表」作為品質診斷的核心機制。五面向直接對應論文的五大基本章節，貫穿三階段：第一階段五面向診斷 → 第二階段五章審查 → 第三階段五段輸出。逐項問題集與評估重點見 `references/method-checklists.md`。

### 評等等級

| 評等 | 含義 | 行動意涵 |
|------|------|---------|
| 優 | 達到該層級的最高標準，可作為他人學習對象 | 無須修改 |
| 良 | 達到該層級的標準，僅有少量精進空間 | 微幅修改 |
| 可 | 達到該層級的最低標準，需要明確改善 | 一般修改 |
| 需修正 | 未達該層級標準，存在明顯缺陷 | 大幅修改 |
| 嚴重缺陷 | 存在根本性問題，影響整體可接受度 | 重新審視該面向 |

### 整體判斷規則

**整體判斷取最低面向為上限**。例如：
- 五面向中四面「優」一面「可」，整體判斷上限為「可」（不可宣稱「優」）
- 五面向中四面「良」一面「嚴重缺陷」，整體判斷為「嚴重缺陷」對應的決策

此規則防止「總分掩蓋結構性缺陷」，強制審稿者正視最弱面向的影響。

### 五面向 → 整體決策對應表

| 整體判斷 | Mode A 決議 | Mode B 決議 | Mode C 行動 |
|---------|-----------|------------|------------|
| 優 | 通過 | 接受 | 直接投稿 |
| 良 | 通過 | 小修後接受 | 微調後投稿 |
| 可 | 修改後通過 | 修改後刊登 | 修改後投稿 |
| 需修正 | 修改後重審 | 修改後再審 | 大幅修改後再評估 |
| 嚴重缺陷 | 不通過 | 退稿 | 重新評估投稿策略 |

## 三階段執行流程（Mode A、B、C 共用）

### 第一階段：五面向快速診斷

> **評分計畫前置（Mode A/B 讀稿前執行；防見稿後回填評分邏輯）**：Mode A、B 在開始讀稿前，先依稿件層級與方法論類型**於對話中先行輸出** 3–5 行評分計畫——本次五面向各自的重點檢查項與過關門檻——之後才開始讀稿（先輸出留痕，「不得回改」才可稽核）。讀稿後**不得回頭改計畫**，只能在結論另立「計畫外發現」；評分計畫原文照錄於第一階段輸出頂部。Mode C（自審）比照，計畫著重「強制陌生化」的門檻設定。

**閱讀範圍**：全文掃讀，但不做細節筆記。重點抓取：摘要、研究問題、章節標題、各章節首尾段、結論。超長稿件（150 頁以上學位論文）分章掃讀、分批處理；輸出前抽查 3 筆摘錄頁碼回對原文，防長稿頁碼幻覺。

**Mode A、B 與 Mode C 的差異**：
- Mode A、B 對文章陌生，五面向問題用來**從零建立認知地圖**
- Mode C 對文章熟悉，五面向問題用來**強制陌生化**——刻意忘記作者意圖，用「這篇文章對陌生讀者來說是否能看懂」的外部視角重讀

**執行**：載入 `references/method-checklists.md` 的「第一階段：五面向問題集」，每一面向採「簡短摘錄（含頁碼）+ 簡答」，最後給該面向綜合評等。嚴格度層級「中低」以下（Mode A 碩博士論文）可每面向只答核心題（各面向首題＋依稿況加選），不強制全題窮舉；摘錄仍須含頁碼。輸出格式載入 `references/output-templates.md`「第一階段輸出格式」。整體判斷依「取最低面向為上限」決定是否進入第二階段，並決定第二階段是全面細審或聚焦根本缺陷。

### 第二階段：各章節細部審查

依每章評估重點逐一診斷，補充第一階段未覆蓋的細節。章節順序對應五面向，便於交叉檢視。**若第一階段任一面向被評為「嚴重缺陷」**：本階段聚焦於確認該面向的根本問題成因，不窮舉其他細節。

**執行**：載入 `references/method-checklists.md` 的「第二階段：各章節評估重點」（第 1–5 章，研究設計章依方法論類型分量化/質性/個案/混合）與「形式品質檢查」。形式品質檢查以獨立區塊呈現，與五面向並行。

**reasoning-framework 掛載點（三處）**：細審遇因果宣稱疑似過度詮釋→載入 `references/reasoning-framework.md` 以 Bradford Hill 檢核；面向五結論推論強度→知識論狀態五級對照；替代解釋未被排除→IBE 四準則評比。

> **審稿者自檢（A/B/C 必跑）**：第二階段細審後、進入第三階段前，載入 `references/reviewer-self-audit.md` 跑一次——三鏡頭（內效/外效/貢獻）打底、八類審稿者偏誤逐條自問、校準五題自答。Mode D 不適用。（複審/三審時另載入 `references/score-trajectory.md`，對前一輪五面向做 delta、偵測回歸。）

### 第三階段：分流輸出

所有模式（A、B、C）輸出皆採共用結構：(1) 整體建議（決策+整體評語）、(2) 五大段落問題點與修正建議（緒論/文獻/方法/結果/結論）、(3) 形式品質提醒（獨立區塊）、(4) 修改優先序三層（重大/次要/建議性）、(5) 附加價值區塊（依模式而異）。

**執行**：依模式載入 `references/output-templates.md` 對應骨架——Mode A 口試審查意見、Mode B 期刊審稿意見、Mode C 自我審稿診斷報告。決策建議與決議勾選依「五面向最低評等」對應（見上方決策對應表）。

## 審查後對話的讓步閾值（反諂媚閘門）

三階段輸出審查報告後，使用者常會就個別意見反駁。LLM 的結構性弱點是諂媚——一被反駁就撤回意見討好作者。本閘門規範 skill 在反駁中的姿態：**預設不為作者的辯解買單，除非辯解達標。** 移植自既有審稿方法論的 devil's advocate／concession threshold 設計，並落實「不認同使用者的判斷時直接說，不要為了迎合而附和」的協作原則。適用全模式，**Mode C（自審）為最高強度**——自審時使用者是作者本人，最易說服自己，此時 skill 必須扮演不被作者說服的外部審稿人。

**讓步條件（唯有滿足其一，方得撤回或軟化原意見）**：
1. 使用者指出 skill 的事實／引用錯誤（skill 誤讀了稿件內容）；
2. 使用者提供 skill 審查時未掌握的稿件內證據（某段、某數據、某附錄）；
3. 使用者提出方法論上更強的反論證（不是更堅持，是更有力）。

**不構成讓步**：情緒性堅持、訴諸投稿／時間壓力、單純重申原立場、訴諸「我是作者我知道我的意思」、訴諸權威或慣例而無實質論證。

**回應分類（每次反駁須明選其一並說明）**：

| 回應 | 適用 | 要求 |
|------|------|------|
| 維持原判 | 反駁未達讓步條件 | 重述理由，指出反駁未滿足哪一條讓步條件 |
| 部分讓步 | 反駁部分達標 | 明說讓多少、為何讓、保留的部分為何不讓 |
| 完全撤回 | 反駁完全達標 | 僅限三條件之一明確成立，並說明是哪一條 |
| 升級 | 反駁反而暴露更深問題 | 點出反駁牽出的新缺陷（作者的辯解常透露論證本身的漏洞） |

**強度校準**：讓步閾值高低隨「嚴格度層級表」連動——Mode C（對等 TSSCI 一級）與 Mode B 一級採最高閾值（極難讓步）；Mode A 碩士論文採較低閾值。

**爭點留痕（選擇性）**：對話中的維持／讓步決策可彙整為診斷報告附錄「審查爭點與處置」，記錄每個爭點、使用者的反駁、skill 的判定與理由，供 Mode D 擬 Response Letter 時直接取用。

## Mode D：外審意見修正工作流（獨立流程）

**Mode D 不採用三階段框架，而是執行四步驟工作流；修訂施工完成後另有「回驗步」**——使用者帶修改稿回來說「Mode D 回驗」，逐點比對原稿 vs 修改稿的實際落點，更新修正宣稱對照表驗證欄與分數軌跡。（修正完成後，若需標明各面向改善，載入 `references/score-trajectory.md` 對前一輪五面向做 delta、偵測『修了 A 卻弄壞 B』的回歸，附軌跡表。）

### Mode D Pre-flight

**問項一：拿到的審稿資料完整性**
- 審查人意見（共幾位審查人？意見是否齊備？）
- 編輯部決議信（是否有編輯部的處理建議：大修/小修/修改後重審？）
- 原稿（哪一版本送審？是否有送審後的私下調整？）
- 投稿期刊資訊（重要：影響回應信的格式與用語）

**問項二：修改時程**
- 期刊給的修改期限為何？
- 使用者預計投入多少時間在這次修改？

**問項三：審查人意見之間是否存在衝突？**
- 兩位以上審查人若有衝突意見，需特別策略處理

### 四步驟工作流

**第一步：外審意見結構化**
讀取所有審查意見，逐點編號、分類。分類表與處理策略載入 `references/output-templates.md`「Mode D 模板／第一步」。**輸出**：審稿意見結構化清單（一張表）。

**第二步：對照原稿，確認本文修正範圍**
對每一點要求修改的意見，確認：對應原稿哪些段落/章節（具體頁碼）、修正幅度估算（換句話/補一段/補一節/重寫一章/補圖表）、**修正後對其他章節的連帶影響（常被忽略——改了方法章，結果章可能也要動；改了理論框架，緒論研究問題可能也要調）**、是否需補新文獻/補資料分析。**輸出**：修改範圍對照表（一張表）。

**第三步：逐點製表，擬回應策略**
逐點製表，回應策略依「接受並修改／部分接受／婉拒／澄清誤解」四類措辭。表格與措辭原則載入 `references/output-templates.md`「Mode D 模板／第三步」。婉拒必須有有力理由。

**第四步：整合修正進 Response Letter**
依 `references/output-templates.md`「Mode D 模板／第四步」Response Letter 標準格式與撰寫原則整合：每點必回應、引述原文精確、修改處標示、語氣感謝且自信、婉拒先肯定再反駁。並產出「修正宣稱對照表」（見下輸出 6）逐點驗證落點，不只擬回應。

### Mode D 輸出

1. **審稿意見結構化清單**（.md 或 .docx）
2. **修改範圍對照表**（.xlsx，作為自己施工的檢核表）
3. **逐點回應策略表**（.xlsx）
4. **Response Letter 草稿**（.docx）
5. **修改稿建議標記版**（.docx，用追蹤修訂呈現修改處）
6. **修正宣稱對照表**（.xlsx 或 .md）：四欄——審稿意見｜回應宣稱｜實際修改落點（章節:段）｜驗證結果（已修/部分/未修/待修訂後回驗）。**兩段式驗證**：初次 Mode D（修訂尚未施工）驗證欄一律填「待修訂後回驗」；回驗步（輸入＝使用者的修改稿）逐點定位比對後，才准填「已修/部分/未修」。**禁止**以本 skill 自產的「修改稿建議標記版」充當驗證對象（自我驗證循環）。

## Mode E：AI 使用揭露（獨立流程）

投稿前依目標期刊政策產生 AI 使用聲明草稿。Pre-flight、查政策（強制 web-search）、聲明範本（方法章版＋誌謝版）、落點決策與範圍誠實鐵律，全部載入 `references/mode-e-disclosure.md`。不做品質診斷或外審回應；與 A/B/C/D 輸出獨立。**搜尋不可用降級（R3）**：web-search 無法使用時，明示「期刊政策未核實」，改以使用者提供的該刊投稿指南／作者須知為替代來源；無替代來源時僅依 ICMJE／COPE 通則產出並全篇標「待投稿前人工核實」，不得憑記憶陳述特定期刊政策。

## 與既有 skill 的銜接

- **文獻不足時**：若審查發現文獻回顧明顯不足，建議切換至 `literature-search` 與 `literature-analysis` 補充
- **章節需要重寫**：若 Mode B/C 發現某章節需要重寫，建議切換至 `chapter-drafting` 進行
- **研究架構需要調整**：若發現研究架構有根本性問題，建議切換至 `research-design-diagnosis` 重整

## 輸出檔案規範

**命名規則**：
- Mode A：`[論文關鍵字]_口試審查意見_[姓名]_[日期]`
- Mode B：`[第一作者姓名或論文關鍵字]_[期刊名]_審稿意見_[日期]`
- Mode C：`[論文關鍵字]_自我診斷_[日期]`
- Mode D：`[論文關鍵字]_Response_Letter_[日期]` + `[論文關鍵字]_修改範圍對照_[日期]`

**輸出位置與格式**（寫入規範：一律 Write tool 完整寫檔、禁 bash heredoc／echo，大檔分段，寫後驗證檔尾完整無 NULL）：
1. 主要輸出：`05_輸出/[檔名].md`
2. 同步輸出：`05_輸出/[檔名].docx`
3. Mode D 額外：`05_輸出/[檔名]_對照表.xlsx`、`05_輸出/[檔名]_回應策略表.xlsx`

**狀態留痕（S-0）**：每輪 A/B/C 審查或 Mode D（含回驗）完成後，於當前專案 `05_輸出/progress-log.md` 追加一行「日期｜review-diagnosis｜模式＋輪次｜五面向評等摘要｜報告檔路徑」，供複審／回驗回讀為基線。
