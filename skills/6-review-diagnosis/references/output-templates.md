# review-diagnosis｜輸出模板參考（output-templates）

> 載入時機：第一階段輸出（診斷格式）與第三階段分流輸出（Mode A/B/C 各自格式），以及 Mode D 第三、四步的表格與 Response Letter。SKILL.md 守決策與流程判斷，本檔提供逐字輸出骨架，於產出階段才載入。範本與本檔衝突時：格式、語調、深度以範本為基準；規格與流程以 SKILL 條文為準。

---

## 第一階段輸出格式（五面向快速診斷）

```
【第一階段：五面向快速診斷】

嚴格度層級：[依 Pre-flight 確認的層級]
方法論類型：[量化/質性/混合，或「待第二階段確認」]
評分計畫（讀稿前定稿，照錄）：
[3–5 行：五面向各自的重點檢查項與過關門檻]

────────────────────────────────────────
面向一：價值定位｜評等：[優/良/可/需修正/嚴重缺陷]
────────────────────────────────────────
Q1.1 ……
  摘錄：「...」(p.X)
  簡答：……

[Q1.2–Q1.5 同格式]

【面向一總結】：[一句話判斷]

────────────────────────────────────────
[面向二、三、四、五 同格式]
────────────────────────────────────────

【計畫外發現】：[讀稿後才發現、不在評分計畫內的檢查點；無則寫「無」]

【整體判斷】：[取最低面向為上限]
最弱面向：[哪個面向]
最強面向：[哪個面向]
是否進入第二階段：[是/否]
  - 若所有面向皆「優」或「良」：第二階段做全面細部審查
  - 若任一面向「嚴重缺陷」：第二階段聚焦於確認該面向的根本問題成因，不窮舉其他細節
```

---

## 第三階段共用結構（Mode A、B、C）

所有模式（A、B、C）的輸出皆採以下結構：

1. **整體建議**（決策 + 一段整體評語）
2. **五大段落問題點與修正建議**（依緒論 / 文獻 / 方法 / 結果 / 結論順序）
3. **形式品質提醒**（獨立區塊）
4. **修改優先序分層**（重大 / 次要 / 建議性 三層）
5. **附加價值區塊**（依模式而異）

## Mode D 模板

### 第一步：外審意見結構化分類表

| 分類 | 處理策略 |
|------|---------|
| 要求修改（重大） | 必須修改，需要在稿件中具體調整 |
| 要求修改（次要） | 通常需要修改，視合理性決定 |
| 要求修改（編輯性） | 文字、格式調整 |
| 請求補充說明 | 不必改稿，只需在 Response 中解釋 |
| 審查人誤解 | 在 Response 中澄清，必要時在稿件中加註避免再被誤解 |
| 審查人之間意見衝突 | 選邊或折衷，需要明確說理 |
| 婉拒修改 | 必須提供有力的反駁理由（例如方法論限制、文獻支持等） |

### 第三步：逐點回應策略表

| 編號 | 審查人 | 原意見摘要 | 本研究的回應策略 | 對應修改位置 | 修改內容摘要 | 修改幅度 |
|------|--------|-----------|----------------|------------|------------|---------|
| R1.1 | A | …… | 接受/部分接受/婉拒+理由 | 第二章 p.X | 補一段討論…… | 補一段 |

**回應策略的措辭原則**：
- **接受並修改**：「我們同意審查人的意見，已於……（位置）進行修改，新增……。」
- **部分接受**：「我們認同審查人的觀點，但考量……，本研究採取折衷方式：……。」
- **婉拒**：「感謝審查人的建議。考量……，本研究維持原有處理方式，理由如下：……。我們已於研究限制中加註此一議題。」（婉拒必須有有力理由）
- **澄清誤解**：「感謝審查人的提問。為避免可能的誤解，我們已於……（位置）補充說明……。」

### 第四步：Response Letter 標準格式

```
[期刊名稱] Response to Reviewers
論文編號：……
論文題目：……

Dear Editor and Reviewers,

We thank the editor and the reviewers for their constructive comments and suggestions, which have helped us significantly improve the manuscript. We have carefully revised the manuscript according to all comments. Below please find our point-by-point responses, with the relevant changes highlighted in the revised manuscript.

Sincerely,
[作者]

────────────────────────────
Response to Reviewer 1

R1.1 [審查人原意見原文摘錄]
Response: [我們的回應 + 修改說明]
Manuscript change: [修改處摘錄，標明位置]

R1.2 ...
────────────────────────────
Response to Reviewer 2
R2.1 ...
────────────────────────────
Editor's Comment（若有）
E1 ...
```

**Response Letter 撰寫原則**：
- 每一點都必須回應，不可遺漏
- 引述審查人原文時務必精確（避免被審查人覺得「我的意見被斷章取義」）
- 修改處用粗體、底線或顏色標示，與內文摘錄並列
- 整體語氣感謝且自信，避免過度卑屈或過度抗辯
- 婉拒的點要先肯定審查人的關切，再提出反駁
