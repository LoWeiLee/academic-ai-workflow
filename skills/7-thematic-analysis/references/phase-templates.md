## 啟動宣告（Pre-flight 完成後）
```
【thematic-analysis 啟動就緒】
資料集：[來源與份數，例：公共服務數位化基層經驗訪談 12 份，p01–p12]
路線：[reflexive TA / coding reliability（codebook）]
知識論立場：[realist / contextualist / critical realist / constructionist]
編碼取向：[inductive / deductive（理論：＜你的理論框架＞）/ hybrid]
分析深度：[semantic / latent / both]
編碼者結構：[單一研究者 researcher-as-instrument / N 名獨立編碼者]
去識別化硬閘門：[已確認去識別化且同意涵蓋 AI 處理 / 未確認—拒絕啟動]
一致性守門：[通過 / 命中：XXX，已請使用者修正]
互動模式：階段逐一確認（每階段產出後等確認再進下一階段）
請使用者確認設定，進入階段 1。
```

## 編碼簿格式
| code | 定義 | 取向(I/D) | 範例摘錄(節略) | 來源(檔:位置) | 階段演化註記 |
|------|------|----------|---------------|---------------|-------------|

## 引文溯源表（防 qualitative deepfake 的核心產物）
| 摘錄ID | 直接引文（逐字） | 受訪者(化名) | 來源(檔:段/行) | 支撐的 code/theme | 回溯核對(✓/✗) |
|--------|-----------------|-------------|----------------|-------------------|---------------|
> 規則：✗ 者必須刪除，不得作為支撐證據；節略以 [...] 標示；不得改寫後仍以引號呈現。

## 主題定義表
| 主題 | central organising concept（一句） | 定義與邊界 | 它說的故事 | 與理論框架的對話 | 涵蓋 code | 與其他主題的關係 |
|------|-----------------------------------|-----------|-----------|------------------|----------|------------------|

## 主題地圖（擇一：文字階層或 mermaid）
文字階層：
```
主題 A：[名稱]
  ├─ 子主題 A1：[名稱]
  └─ 子主題 A2：[名稱]
主題 B：[名稱]
```
mermaid（如需視覺化）：
```
mindmap
  root((研究問題))
    主題A
      子主題A1
      子主題A2
    主題B
```
