# route-coding-reliability｜coding reliability／codebook 路線（骨架版）

> 載入時機：Pre-flight 定路為 coding reliability／codebook 後載入。
> **狀態：骨架版。** 已列齊與 reflexive 路線的分岔決策點與信度程序框架，逐階段操作細節待後續版本依使用經驗補完。若使用者現在就要走此路線：**骨架模式產出邊界**＝研究設計備忘＋codebook 草案為止；信度係數計算（kappa／α）暫不提供，不得即興生成計算程序與係數值——骨架產出頂部標記【骨架模式：信度程序未經驗證，不得寫入投稿稿件】。

## 路線前提
- 編碼者間信度（ICR）是嚴謹度的核心證據；目標是降低編碼者主觀性對結果的影響。
- 編碼簿預先建立、早期鎖定；編碼者依固定編碼簿獨立編碼。
- 知識論偏 realist／post-positivist；撰寫可用「辨識／浮現主題」語言。

## 與 reflexive 的分岔決策點（必須對齊的設定）
1. **編碼者**：≥2 名獨立編碼者；明確分工與獨立編碼範圍。
2. **編碼簿**：預先固定／早期鎖定；定義、納入排除準則、範例摘錄齊備。
3. **信度指標**：Cohen's kappa（兩編碼者、名目）／Krippendorff's α（多編碼者、可處理缺失與多尺度）＋ per-theme % agreement。報告係數值與判讀（如 α≥.80 可接受，視領域而定）。
4. **歧見處理**：記錄編碼歧見、協商解決（consensus）或仲裁程序。
5. **主題性質**：編碼者共識的主題／領域桶。
6. **撰寫用語**：信度、透明度、可複製；可用「主題浮現／被辨識」。

## 信度程序框架（待後續版本補操作細節）
- 編碼簿建立 → 試編一小批 → 計算初步信度 → 修訂編碼簿至穩定 → 正式獨立編碼 → 計算最終信度 → 歧見協商 → 鎖定。
- 工具層面（ATLAS.ti／NVivo 的 ICR 模組或自寫腳本）細節待補。

## 一致性守門提醒
本路線若退化為單一編碼者、或撰寫採 reflexive 知識論語言，即與後實證前提矛盾——此時應回 Pre-flight 重新確認是否其實該走 reflexive 路線。
