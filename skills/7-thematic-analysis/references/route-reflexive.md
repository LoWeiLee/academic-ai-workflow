# route-reflexive｜reflexive TA 路線操作（Braun & Clarke）

> 載入時機：Pre-flight 定路為 reflexive TA 後，全程載入。本檔提供六階段的逐階段操作與 reflexive 特有紀律。SKILL.md 守決策層，本檔守路線操作層。

## 路線前提（貫穿全程）

- **編碼者主觀性是分析資源，不是要消除的雜訊。** 你的理論視角、學科訓練、實務背景，都是詮釋深度的來源。
- **不計算、不報告 inter-coder reliability（kappa／Krippendorff's α）。** 在 reflexive TA 中補信度係數是典範混用，會被審稿人視為方法論不一致。嚴謹度改由反身性、審計軌跡、理論透徹度承擔。
- **主題是被「建構／發展」出來的，不是「浮現／被發現」的。** 撰寫用語全程避免「themes emerged」這類被動發現語言。
- **編碼簿是演化的工作文件，不預先固定。** 容許編碼隨對資料的理解加深而修改、合併、重命名。

## 知識論宣告（Pre-flight 已選，此處落實）

依宣告的立場校準詮釋姿態：realist／essentialist（貼近受訪者明說的 semantic 意義）、contextualist／critical realist（意義受脈絡中介）、constructionist（意義由社會建構，分析挖掘其生產條件）。立場決定 semantic vs latent 的比重，並貫穿撰寫用語。

## 反身性紀律

全程維護一份「反身性紀錄」（reflexivity journal，產出於 05_輸出/），記錄：分析者的預設立場與理論透鏡如何影響編碼判斷、關鍵詮釋決策的轉折點、與既有理論（如＜你的理論框架＞）的對話。此紀錄是 reflexive TA 嚴謹度的核心證據，也供 review-diagnosis 面向三檢核。

## 六階段操作

> **逐場處理協議（token 護欄，全程強制）**：任何「通讀全部」「對照整個資料集」的要求，一律以**逐場循環**實作——讀入一場逐字稿→產出該場筆記／編碼→寫檔（05_輸出/）→釋放，再讀下一場；**嚴禁全集同時駐留 context**。跨場比對（階段 3–4）以各場已落檔的筆記／編碼表為輸入，需要原文時定向回讀該場該段（工作副本有行號）。資料量大（>10 場）時，階段 1–2 可評估以 subagent 分批（每個 3–5 場）平行處理後彙整。

### 階段 1｜熟悉資料
通讀全部逐字稿至少一遍，產出每份的沉浸式筆記（初步印象、反覆出現的概念、與研究問題的關聯、令你意外之處）。不編碼。完成後交使用者確認「熟悉度筆記」。

### 階段 2｜產生初始編碼
逐段編碼，code 名要具體、承載意義（避免空泛標籤）。inductive 時讓 code 貼資料；deductive／theoretical 時以 paper-structure.md 的理論框架（＜你的理論框架＞）為編碼透鏡，但允許框架外的資料驅動 code 並標記。每個 code 即時登錄引文溯源（檔名＋位置）。產出初版編碼簿＋引文溯源表。

### 階段 3｜建構初始主題
將 code 聚類為候選主題——主題是「環繞核心概念的共享意義模式」，不是 code 的容器。為每個候選主題寫一句 central organising concept。產出候選主題清單＋初版主題地圖。

### 階段 4｜發展與檢視主題
雙層檢視（Braun & Clarke 的兩層次）：(1) 對照各主題的編碼摘錄是否連貫；(2) 對照整個資料集，主題是否站得住、是否漏掉重要模式。檢查主題內部同質、主題間區隔。可合併、拆分、捨棄。記錄每個調整的理由於反身性紀錄。

### 階段 5｜精煉、定義、命名主題
為每個主題寫：定義（範圍與邊界）、它說了什麼故事、與其他主題的關係、命名（精準、可讀、承載分析）。產出主題定義表。確認無「主題其實是兩個主題」或「主題只是一個 code」的問題。

### 階段 6｜撰寫（交付 chapter-drafting）
為每個主題寫敘事骨架：核心論點、支撐摘錄（已過引文溯源稽核）、與理論框架的對話、實務意涵（扣使用者的實務落地維度）。**不寫最終正文**——交付給 chapter-drafting，由其依 writing-standards 落成結果／討論章。交付前執行引文溯源全量回溯核對。

## reflexive 路線常見失誤（自檢時排查）
- 把 code 當主題（主題太薄、只是貼標籤）。
- 主題之間重疊不清（階段 4 區隔沒做足）。
- 撰寫混入「emerged」「inter-coder agreement」等不一致語言。
- 宣稱統計代表性或頻次即重要性（質性不以次數定主題分量）。
